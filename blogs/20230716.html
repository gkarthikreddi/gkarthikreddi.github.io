Fixing Unverified Git Commits

July 16, 2023

If you are getting unverified tag after commiting to github using ssh on your machine then most probably you need to fix the following things

  • Check if you have same ssh key in user.signinkey in git config and make sure that the same ssh key is uploaded to the github as Signing key
  • Check if your github email is linked to the user.email in git config

First check your git config by entering the following command in terminal:
> git config --list

If you don't have user.signinkey then add it to the config:
> git config user.signinkey = YOUR_SSH_PUBLIC_KEY

Make sure you upload the same key to your github account
settings--> ssh and gpg keys--> add new key (choose the key type as Signing key)

If you don't have user.email then add it to the config:
> git config user.email = YOUR_EMAIL

You can also do this by changing the .gitconfig file. If don't have .gitconfig in your home directory then create one and add the following things.
# .gitconfig
[user]
email = YOUR_EMAIL
name = YOUR_NAME
signingkey = YOUR_SSH_PUBLIC_KEY

Once again check gitconfig by:
>> git config --list

After adding both ssh key and email to git config your commits should have a verified tag, if you did everything right.
